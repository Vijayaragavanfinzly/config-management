How Sync Works

A step-by-step breakdown of how the Sync process functions in our application.

Step 1: Start Sync Process

The sync process begins when the user clicks the Sync button in the application. This initiates the retrieval of data from multiple environments.

Step 2: Fetch Data from Config Servers

API calls are made to the following config servers:

  • dev
  • dev2
  • qa
  • qa2

Properties are retrieved from their respective environment tables.

Step 3: Process Properties

Each property fetched from the servers is processed individually. The system determines if the property is tenant-based by matching it with active tenants from bankos.tenants.

If tenant-based: The tenant name is assigned to the property.

The processed properties are transformed and prepared for storage.

Step 4: Store Processed Data in Local DB

All validated and processed properties are stored in the local database under a unified structure called Processed Properties. This includes both tenant-based and other properties.

Step 5: End Sync Process

The sync process is completed. All fetched and processed properties are now available in the local database for further comparison and operations.
